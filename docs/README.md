# 项目文档目录

本目录用于承载仓库的扩展说明。根目录 README 后续可以作为项目入口页，本目录中的文档则用于解释背景、架构、部署、数据和算法细节。

## 文档列表

- [项目背景与数据说明](./project-background.md)：说明毕业设计背景、真实数据来源、数据库缺失原因和隐私边界。
- [总体架构](./architecture.md)：说明前端、后端、大数据、爬虫和数据库之间的关系。
- [数据库说明](./database.md)：说明业务库、Hive 表、核心字段、数据缺失影响和后续重建建议。
- [Linux 与大数据部署说明](./deployment-linux.md)：说明完整运行所需的 Ubuntu/Hadoop/Spark/Hive/Kafka 环境。
- [推荐算法与数据链路](./recommendation-algorithm.md)：说明用户行为采集、Kafka 流转、Spark 计算和推荐结果写入。
- [爬虫子项目说明](./crawler.md)：说明爬虫的定位、技术栈、输入输出和与主系统的关系。

## 当前状态

本仓库目前保留了毕业设计时期的完整工程代码和目录结构整理结果，但不包含原始 MySQL 业务数据与 Hive 运行数据。原因和影响见 [数据库说明](./database.md)。

根目录中的 `database/` 与 `scripts/` 目前仅作为本地预留目录，因 Git 不提交空目录，首次上传到 GitHub 时不会出现；后续补充脱敏 SQL、初始化脚本或部署脚本时再创建对应内容。

后续根目录 README 可以从这些文档中提炼：

- 项目简介
- 功能列表
- 技术栈
- 架构图
- 运行环境要求
- 部署步骤
- 数据缺失说明
- 项目亮点与可改进点
